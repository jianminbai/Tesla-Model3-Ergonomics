# 第十章 可视化插图与图表体系

> 本章核心观点：驾驶人体工程学很难只靠文字讲清楚。坐骨、大腿、骨盆、靠背、前沿、踏板之间的关系，本质上是空间关系和力学关系。可视化插图不是装饰，而是本项目的核心表达方式。

---

## 10.1 为什么本项目必须使用图

座椅人体工程学的问题有三个特点：

1. **空间关系复杂**  
   骨盆、坐骨、大腿、靠背、踏板同时参与。

2. **体感描述模糊**  
   “疼”“压”“紧”“挤”“麻”常被混用。

3. **调整变量耦合**  
   升高座椅可能改善坐骨，也可能增加大腿后侧压力；后移座椅可能减轻大腿根，也可能影响刹车安全。

这些关系如果只靠文字，很容易造成误解。

例如：

```text
座椅升高
↓
坐骨两侧软组织可能减轻
↓
但大腿后侧压力可能增加
```

这句话正确，但不够直观。用图可以同时展示：

- 髋膝关系；
- 大腿接触面积；
- 坐骨压力；
- 脚跟支点；
- 风险区域。

因此，本项目从 V0.7 开始，把插图作为正式内容资产，而不是后期补充。

---

## 10.2 为什么选择 SVG

本项目优先使用 SVG，而不是 PNG/JPG。

原因：

1. **矢量清晰**  
   在 GitHub、网页、PDF 中放大不糊。

2. **便于版本控制**  
   SVG 是文本文件，Git 可以跟踪变化。

3. **便于修改**  
   后续可直接调整标签、颜色、箭头和布局。

4. **适合工程图**  
   本项目大多数图是结构图、流程图、受力图，而不是照片。

5. **可被 MkDocs 直接引用**  
   Markdown 中可以直接插入：

```markdown
![压力分布](../assets/diagrams/svg/01_pressure_distribution.svg)
```

## 10.3 插图类型

本项目插图分为五类。

### 第一类：受力图

用于解释压力如何变化。

例：

- 压力集中 vs 压力分散；
- 座椅高度变化；
- 前沿高度变化；
- 后移 1 cm 的腿部几何变化。

### 第二类：姿态图

用于解释身体姿势。

例：

- 骨盆后倾 / 中立 / 过度前倾；
- 腰背贴合；
- 肩膀是否离开靠背；
- 脚跟支点。

### 第三类：结构图

用于解释 Model 3 座椅特征。

例：

- 坐垫俯视图；
- 侧翼横截面；
- 坐垫前沿；
- 靠背与腰托。

### 第四类：流程图

用于决策。

例：

- 坐骨单点疼决策树；
- 大腿后侧紧硬决策树；
- 保留 / 回退判断；
- 14 天实验流程。

### 第五类：趋势图

用于实验记录。

例：

- 坐骨疼评分趋势；
- 大腿后侧紧硬趋势；
- 座椅调整事件时间线；
- 办公久坐与驾驶不适关系。

## 10.4 统一颜色规范

为了降低理解成本，所有图使用统一颜色语义。

| 颜色 | 含义 | 使用场景 |
|---|---|---|
| 灰色 | 座椅 / 中性结构 | 坐垫、靠背、地板 |
| 蓝色 | 正常压力 / 正常承托 | 大面积分散承重 |
| 橙色 | 压力增高 / 需要观察 | 大腿压力、侧翼包裹 |
| 红色 | 风险压迫 / 回退信号 | 麻刺、单点高压、放射风险 |
| 绿色 | 改善方向 / 可保留 | 合理调整路径 |
| 黄色 | 骨性结构 / 中间提示 | 坐骨、骨盆、说明框 |

这个颜色规范比“画得好看”更重要。读者一看到红色，就知道是风险区域；看到绿色，就知道是建议方向。

## 10.5 插图不应制造医学误导

本项目是人体工程学指南，不是医学诊断工具。因此插图需要避免：

- 画出过于医学化的神经病变结论；
- 暗示某个症状一定是坐骨神经痛；
- 把所有疼痛都画成严重损伤；
- 使用恐吓式红色大面积疼痛图；
- 用“治疗”“治愈”等表达。

更合适的表达是：

- 风险区域；
- 可能机制；
- 优先检查；
- 建议回退；
- 应咨询医生或物理治疗师。

## 10.7 Markdown 引用方式

在章节中引用图片时，建议使用相对路径。

例如在 `book/01-驾驶为什么会让人疼.md` 中引用：

```markdown
![压力不是消失，而是重新分配](../assets/diagrams/svg/01_pressure_distribution.svg)
```

在 `book/02-骨盆决定受力.md` 中引用：

```markdown
![骨盆后倾、中立、过度前倾](../assets/diagrams/svg/02_pelvis_postures.svg)
```

在 `book/04-Tesla_Model3座椅结构分析.md` 中引用：

```markdown
![Model 3 坐垫俯视结构](../assets/diagrams/svg/03_model3_seat_top_view.svg)
```

## 10.8 插图文件命名规则

建议：

```text
编号_英文主题.svg
```

例如：

```text
01_pressure_distribution.svg
02_pelvis_postures.svg
03_model3_seat_top_view.svg
04_seat_height_force_change.svg
```

规则：

- 使用两位数字编号；
- 使用小写英文；
- 单词之间用下划线；
- 不使用空格；
- 不使用中文文件名；
- 一个图表达一个核心概念。

---

## 10.9 后续插图迭代流程

每张图建议遵循以下流程：

```text
提出图的目的
↓
定义图中元素
↓
确定核心结论
↓
绘制 SVG
↓
放入 assets/diagrams/svg
↓
写入 docs/diagram_index.md
↓
在对应章节引用
↓
根据读者反馈迭代
```

不要为了画图而画图。每张图都必须服务一个明确结论。

## 10.10 本章小结

V0.7 之后，本项目进入图文结合阶段。

文字负责解释：

- 为什么；
- 如何判断；
- 如何验证。

图负责展示：

- 受力在哪里；
- 调整改变了什么；
- 哪些区域是风险；
- 哪些路径是推荐方向。

最终目标是让读者不用具备专业人体工程学背景，也能通过图理解：

```text
坐骨为什么会疼
大腿为什么会紧
右腿为什么会累
座椅为什么不能乱调
调整后为什么要记录趋势
```
